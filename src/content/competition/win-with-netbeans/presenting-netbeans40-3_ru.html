J2ME

Приложения на Java могут работать на мобильных телефонах, интерактивных телевизорах; это рай для Java. Корпорация Sun предоставляет комплект разработчика для J2ME. Он интегрируется в NetBeans при установке комплекта для мобильных устройств.

Пакет можно загрузить по ссылке: https://netbeans.org/kb/trails/mobility.html

Создание приложения J2ME

Мы воспользуемся примером проекта, чтобы рассказать о различных функциональных возможностях подключаемого модуля для мобильных устройств:

Выберите команду "Файл" -> "Новый проект" (CTRL+SHIFT+N).
Выберите подкатегорию "Мобильное устройство" -> "Мобильное приложение"
Нажмите кнопку "Далее" и назовите проект "MyJ2ME".
Оставьте параметры, выбранные по умолчанию, и нажмите кнопку "Завершить" .

В NetBeans был использован класс шаблона со следующими элементами:

  • метод startApp();
  • метод pauseApp();
  • метод destroyApp();
  • метод commandAction(Command c, Displayable s);
  • класс расширяет класс MIDlet и реализует интерфейс CommandListener.

Вы можете выполнить проект, который запустится на эмуляторе мобильного телефона.

Настройки проекта

Мы можем создать различные конфигурации, позволяющие использовать проект в разных контекстах.

  • Выберите проект на вкладке "Проект", щелкните правой кнопкой мыши и выберите пункт "Свойства".


Окно конфигурации

  • В верхней части окна выберите в поле со списком пункт "Добавить конфигурацию... ". Введите для конфигурации имя "WithKeyboard".
  • Снимите флажок "Использовать значения конфигурации по умолчанию"
  • Выберите в поле со списком "Устройство" пункт "Устройство Qwerty" и нажмите кнопку "ОК".
Теперь запустим проект с конфигурацией WithKeyboard:
  • Щелкните проект правой кнопкой мыши и выберите пункт "Установить в качестве главного проекта", чтобы данный проект стал главным.
  • Выберите на панели инструментов созданную конфигурацию, как показано на рисунке ниже:


Окно конфигурации

  • Теперь необходимо выполнить проект, выбрав команду "Выполнить" -> "Выполнить главный проект...".
  • Откроется выбранный эмулятор с именем QwertyDevice, на котором будет выполнено программное обеспечение j2me.
Конфигурации позволяют в дальнейшем выбирать уровень маскировки, способ создания файла jar, способ создания документации javadoc и т.д.

J2EE с Sun Application Server

В следующей версии NetBeans будет представлена ключевая функция интеграции с Sun Application Server 8.1. Более того, будет добавлена поддержка EJB и веб-служб. Эти функциональные возможности все еще зарезервированы дорогостоящими средами IDE.

Средство профилирования NetBeans

Целью данного подключаемого модуля является предоставление мощного и гибкого решения для профилирования, тесно интегрированного с рабочим процессом IDE среды. Используя это средство, вы получите ясное представление о работе программного обеспечения java во время его выполнения.

Выпуск планируется на апрель 2005 г., но некоторые возможности доступны уже в бета-версии.

Если вы хотите протестировать их:

  • загрузите модуль по ссылке http://profiler.netbeans.org/index.html.
  • После разархивации необходимо выбрать файл .nbm в меню "Инструменты" -> "Центр обновления" -> "Установить модуль вручную".
  • После установки перейдите на вкладку "Проект", выберите класс проекта, содержащий функцию main(), и щелкните в меню "Профиль" команду "Профилирование файла"...

Примечание.
В бета-версии необходимо использовать виртуальную машину 1.4 java. Следовательно, если у вас установлена версия1.5, необходимо выбрать в компиляторе аргумент source 1.4 или установить версию 1.4. Только второй вариант сработал на моем компьютере.

Разное

NetBeans предлагает широкую палитру подключаемых модулей для поддержки других языков, например C/C++, Фортран и даже UML. Также было выпущено средство профилирования, позволяющее тестировать производительность ваших приложений. Мы не будем рассуждать о них в этой статье, это сделаете вы сами :-). Правда большинство подключаемых модулей доступны для версии NetBeans 3.6, и бесполезны для четвертой версии. Мы надеемся, это скоро будет исправлено.

Список подключаемых модулей: http://plugins.netbeans.org/

Разное

Преимущества:

  • Широкая палитра встроенных функциональных возможностей: xml, документация, базы данных, tomcat...
  • Привлекательность для работы с настольными, мобильными и веб-приложениями java.

Недостатки:

  • Модули J2EE будут выпущены только весной 2005 г. в версии 4.1.
  • Большинство подключаемых модулей недоступны для четвертой версии.

Заключение

NetBeans позволяет программистам легко и просто разрабатывать приложения, принимая на себя выполнение рутинных и второстепенных частей проекта. Используя NetBeans, вы сэкономите время, деньги и избавитесь от удерживания в голове кучи задач. Вы сможете концентрироваться только на коде.
