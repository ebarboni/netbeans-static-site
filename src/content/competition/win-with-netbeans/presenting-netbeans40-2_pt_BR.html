Uma palavra de alô em um aplicativo padrão

Vamos iniciar um projeto padrão simples, um aplicativo que irá imprimir a "Alô Mundo". A seguir, vamos adicioná-lo à interface gráfica.

Execute isso:

Arquivo ---> Novo projeto.
Padrão ---> Aplicativo Java
e nomeie seu projeto como "tutorial".

Escreva isso na função principal:

    System.out.println("Alô Mundo");
A seguir, vamos fazer a compliação e execução:
Build ---> Build do projeto principal...
Executar ---> Executar projeto principal ...

A janela de saída deverá imprimir a "Alô Mundo" junto com outras linhas impressas pelo ant.


Editando as interfaces gráficas

Vamos usar um pouco o editor de interface gráfica. Este tem base no Swing, uma das bibliotecas gráficas Java mais usadas.

Execute:
Arquivo ---> Novo projeto...
Formulários IGU Java ---> Jframe
e nomeie sua classe como "javaGui".

Na aba "Paleta", selecione um Jlabel e coloque-o no Jframe que é desenhado no meio da tela. Na aba "Propriedades", preencha a variável "text" com um valor como "Alô Mundo". Sugerimos não se apressar, pois o Swing é uma biblioteca grande e este editor gráfico irá permitir usar a maioria de suas funcionalidades.

Antes de executar o aplicativo, temos que verificar se nossa classe Jframe será a usada no momento da inicialização. Siga e selecione no menu Arquivo ---> Propriedades do tutorial ... ---> Executar ---> Executando projetos e, no campo da classe principal, insira " tutorial.javaGui " ou use o botão Procurar. Desta forma, seu aplicativo irá iniciar no método principal do tutorial.javaGui.

Inicie o aplicativo e examine a janela "Alô Mundo" (feia não é?):

Aviso:
O editor gráfico do NetBeans usa algumas bibliotecas gráficas que não são fornecidas com o Java. Consequentemente, você terá que fornecê-las com seu futuro build do software com o editor gráfico do NetBeans. Embora, não aconselhamos usar o editor de interface gráfica caso não conheça os conceitos básicos das API do Swing. Na primeira vez, é melhor você mesmo escrever suas interfaces gráficas.


Integração Tomcat:

Os desenvolvimentos Web cresceram no último ano. O Java funciona muito bem neste ambiente. O servidorr de aplicativo Web Tomcat está incluído no NetBeans, facilitando o desenvolvimento. Nesta parte, iremos criar um JSP e um Servlet. O objetivo não é o de entender as tecnologias da Web, mas como o Tomcat pode ser usado através do NetBeans.

Editar um JavaBean

Há outras formas para manipular dados de formulários. O JavaBean é uma destas, que consiste na coleta de valores com a classe Java. Esta classe deve aplicar o Padrão de projeto (usando getters e setters para cada valor).

Caso deseje executar um JavaBean:

  • Coloque o cursor em seu projeto da Web.
  • Arquivo ---> Novo arquivo ...
  • Classes Java -> Classe Java
  • Nomeia a classe Java com o "bean" e seu pacote como "beanPacakge".

Editar um servlet:

Os servlets são classes Java usadas para criar páginas da Web. De fato, as páginas JSP são automaticamente convertidas no servlet para que sejam compiladas.

Vamos criar um formulário usando um servlet.

  • Se você deseja criar um servlet:
  • Arquivo ---> Novo arquivo ...
  • Selecione o menu Web ---> Servlet
  • Nomeie o servlet como "destination_servlet" e o pacote como "servletPackage"

O destination_servlet deveria se parecer com isto:

destination_servlet.java

Se abrirmos o WEB-INF/web.xml, poderemos ver o servlet já configurado e pronto para funcionar.

E, ainda usaremos um formulário JSP:

send_servlet.jsp

Clique com o botão direito do mouse no projeto, na aba "Projeto", e selecione "Executar projeto". O navegador da Web irá abrir o arquivo index.jsp. Você irá clicar no terceiro link para testar a solução com base no servlet.
