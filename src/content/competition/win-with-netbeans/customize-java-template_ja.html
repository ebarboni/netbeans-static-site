NetBeans の Java テンプレートをニーズに合わせてカスタマイズする

Win With NetBeans コンペティションの一部として Arthur Cheng 氏から寄稿

NetBeans には汎用的なクラステンプレートが付属しています。ところが、新しいクラスを作成しようとするたびに、汎用的な同じテキスト (会社の著作権の文や免責事項など) を入力し直したりカット＆ペーストしたりしていることに気づくことも少なくないと思います。

その情報をすでに含んでいる Java クラスを NetBeans に作成させることができれば、便利ではないでしょうか。実際、それはきわめて簡単に行えるのです。カスタマイズされたテンプレートがその解決策です。

会社の情報を含むカスタマイズされた Java クラステンプレートを作成する必要があるとします。その設定手順を次に示します (この手順はカスタマイズされた Java インタフェースやアプレットなどにも適用可能です)。それでは始めましょう

  1. 「ツール」>「オプション」を選択します。「オプション」ウィンドウで、「ソースの作成と管理」>「テンプレート」>「Java クラス」>「Java クラス」に移動します。
  2. 「Java クラス」を右クリックして「コピー」を選択します。
  3. 「ソースの作成と管理」>「テンプレート」>「Java クラス」に戻ります。
  4. 「Java クラス」を右クリックして「ペースト」>「コピー」を選択します。
  5. これで、「Class_1.java」がテンプレートの 1 つとして、「Java クラス」の下に表示されるはずです。
  6. 「Class_1.java」を右クリックして「名前を変更」を選択すると、次のウィンドウが表示されます。名前を会社名 (ここでは XYZCompany を使用) に変更し、「了解」をクリックします。(テンプレート名の末尾を単語 Template にしないでください。新しいテンプレートにそのような名前を付けると、テンプレートが NetBeans に表示されません。)

    次のポップアップウィンドウで「次へ」をクリックします。

    するとこのように、新しい Java クラステンプレートが表示されます。

  7. この時点で、名前 Class_1.java が XYZCompany.java に変更されているはずです。ここまでで、作業はまだ半分しか終わっていません。
  8. 「XYZCompany.java」を右クリックして「開く」を選択します。ソースエディタでテンプレートのソースが開きます。「オプション」ウィンドウで「閉じる」をクリックしてウィンドウを閉じ、ソースエディタに移動します。
  9. ここからが魔法の始まりです。ソースエディタからソースコードを好きなように変更します。次の例を参照してください (クラス宣言は変更しないでください。ほかのクラス名をすべて __NAME__ マクロに置き換えておくと、テンプレートからファイルを作成する際にそれらが実際のクラス名に置き換えられます)。
    /*
 * __NAME__.java
 *
 * 作成日時: __DATE__, __TIME__
 *
 * Copyright(c) {YEAR!!!} XYZ Company, Inc.  All Rights Reserved.
 * このソフトウェアは XYZ 社の商用情報です。
 *
 */

package Templates.Classes;

import org.xyz.whatever         //  大部分のクラスに共通する
                                //  インポートを配置します
import org.xyz.util.Logger;     //  ロギングライブラリなど。



/**
 * {クラスの説明をここに挿入します}
 *
 * @version $Revision: 1.1.1.1 $
 * @since Build {バージョンをここに挿入します} (MM YYYY)
 * @author John Doe
 * @author $Author: nbweb $, (このバージョン)
 */
public class XYZCompany
{
     private static final String PVCS_ID = "@(#)$Header: /cvs/testwww/www/competition/win-with-netbeans/customize-java-template.html,v 1.1.1.1 2009/10/29 15:04:23 nbweb Exp $";
    private static final Logger log = Logger.getLogger (__NAME__.class);
    private static final String CLASS_NAME = "__NAME__";
    
    
    
    /**
     * __NAME__ の新しいインスタンスを作成します
     */
    public XYZCompany ()
    {
    }

    
    
    /**
     *  その他のメソッドをここに追加します ...
     *
     *  @param value1
     *  @param value2
     *
     *  @return
     *
     *  @throw
     */
//    public void helloWorld()
  
//    throws whateverException
//    {
//    }
    
}    //  __NAME__ の終了
  10. 終わったら、「ファイル」>「保存」をクリックします。
  11. これで、Java クラスを作成する必要が生じるたびに、新しい XYZCompany Java クラスの作成を選択するだけで、カスタマイズされたすべての情報がそこに含まれるようになります。

デフォルトユーザー名やその他のマクロを設定する

  1. 「ツール」>「オプション」を選択します。「オプション」ウィンドウで、「編集」>「Java ソース」を展開します。
  2. 右側の「プロパティー」タブの「文字列テーブル」の値の省略記号 (...) ボタンをクリックします。
  3. 「USER=ユーザー名」と設定します。
  4. {Macro Name}={Macro Value} の形式でその他のマクロを追加し、「了解」をクリックします。(マクロを定義する際に、マクロ名の先頭や末尾でアンダースコアを使用しないでください)。
