Введение:

Интегрированная среда разработки (IDE) — это набор вспомогательных средств для программистов. NetBeans — одно из таких средств, специально приспособленное к Java. Оно позволяет разгрузить мозг и сосредоточиться на самом коде.

Сначала рассмотрим большую часть функциональных возможностей NetBeans, а затем перейдем к учебным курсам. Наконец, обратимся к подключаемым модулям и преимуществам и недостаткам среды IDE.

NetBeans разрабатывается корпорацией Sun Microsystems и доступен для свободной загрузки на официальном веб-сайте www.net-beans.org.

Предупреждение. Цель этой статьи заключается не в перечислении всех функциональных возможностей, а в предоставлении обзора возможностей. В случае возникновения проблем рекомендуется ознакомиться с официальным веб-сайтом и документацией.

История

1996: запуск проекта NetBeans в чешской компании Xelfi. Планировалось написать подобную Delphi среду IDE для Java и на Java. Вышло две коммерческие версии "Developer" 2.0 и 2.1.

Май 1999 г.: выпуск бета-версии 3.0.

Октябрь 1999 г.: NetBeans приобретается корпорацией Sun Microsystems. Название среды IDE изменяется на "Sun Forte For Java Community Edition". Эта версия бесплатна, а профессиональная продается.

Июнь 2000 г.: корпорация Sun выпускает NetBeans с открытым исходным кодом (под прежним именем).

Декабрь 2004 г.: выпуск четвертой версии с поддержкой Java 5, усовершенствованным диспетчером проектов, усовершенствованным графическим интерфейсом, поддержкой Ant в проектах и т.д.

Планы на будущее:

Весна 2005. Выпуск 4.1 с модулями J2EE

В политике корпорации Sun NetBeans представляет собой ключевое приложение в "Sun ONE Studio" — среда IDE ONE (One Net Environment). В рамках проекта ONE должна быть создана группа приложений для специалистов по сетевым технологиям и разработке ПО. В этом проекте фигурируют "Sun ONE Application server", "Sun ONE Web Services platform" и т.д.

NetBeans работает на ОС Win32, Linux, MacOS X и Solaris. Архитектура полностью написана на Java.

Кроме того все инструменты бесплатны. NetBeans — это складной ножик программиста.

Диспетчер проекта
Рассмотрим пункт меню Файл -> Новый проект

В NetBeans предусмотрены готовые к использованию шаблоны проекта:

  • для стандартных приложений (приложений командной строки и с графическим интерфейсом);
  • для веб-сайтов. В версии 4.1 будут добавлены шаблоны для J2EE, такие как EJB и веб-службы.
  • Пункт меню "пример" с примерами кода
Шаблоны проектов упрощают работу, поскольку NetBeans автоматически настраивает программное обеспечение, например, веб-серверы (Tomcat, который будет рассмотрен позднее).

Создадим проект Стандартный -> Приложение Java с названием "NetBeans".

Создаются два пакета:

"Пакеты с исходными файлами" содержат исходный код. В них размещаются файлы, изображения и т.д.

"Пакеты тестов" содержат исходный код файлов Junit. Junit представляет собой инфраструктуру тестирования (будет рассмотрена позже).

Предупреждение: эти пакеты — не то же самое, что пакеты языка Java.

Если в программное обеспечение требуется добавить элемент, выберите проект на вкладке проектов, а затем выберите пункт меню Файл -> Новый файл...

В этом окне приведено множество шаблонов файлов от пакетов Java до сервлетов и графических интерфейсов Swing.

Поддерживаемые файлы

В среде IDE должны поддерживаться часто используемые типы файлов. Среда IDE NetBeans способна распознавать структуру этих файлов и выделять цветом определенные слова и синтаксис:

  • файлы Java с выделением цветом ключевых слов и синтаксиса, автоматическим дополнением и исправлением ошибок;
  • файлы xml с выделением цветом синтаксиса, генератором dtd и xsl, а также валидатором;
  • файлы html и css с выделением цветом синтаксиса.

Организация файлов

На рисунке показаны файлы в разных каталогах:

  • в каталоге build расположены файлы классов;
  • в каталоге dist расположены итоговые файлы проекта: файл jar приложения и документация;
  • в каталоге nbproject расположены файлы настройки, определяющие компиляцию, выполнение и развертывание проекта;
  • в каталоге src расположены исходные файлы проекта;
  • в каталоге test расположены файлы теста проекта (Junit рассматривается ниже);
  • прочие файлы: build.xml, используемый для компиляции (это файл ant) и manifest.mf для определения проекта.

Формирование файлов документации

Создание документации позволяет другим программистам добиваться понимания архитектуры разрабатываемого кода. Для этого можно воспользоваться средством документации, представляемым в JDK, посредством графического интерфейса NetBeans.

Средство документации формирует файлы документации из комментариев в коде (с такими символами, например, как /** @author имя_автора */)

Для формирования документации достаточно выбрать пункт меню Построение -> Создать документацию проекта. С документацией можно ознакомиться в файле dist/javadoc в папке проекта.

Автоматическая компиляция

За компиляцию отвечают файлы XML, интерпретируемые средством Ant. Если нет особых требований, этот файл генерируется автоматически.

Компиляция выполняется автоматически при выполнении приложения из меню Выполнить -> Выполнить главный проект. Компиляцию можно запустить вручную путем выбора Построение -> Построить главный проект. В подкаталоге dist в корневом каталоге проекта будет создан файл .jar приложения (который также отображается на вкладке файлов).

Файлы XML, предназначенные для средства Ant, автоматически генерируются в файлах "nbproject" в корневом каталоге приложения. Эти файлы используются при компиляции, формировании документации и запуске.

Если читатель не знаком с принципом действия средства Ant, рекомендуется обратиться к официальной документации.

"Отладка"

При написании кода важно держать под контролем все составляющие выполнения программы. Запуск приложения в пошаговом режиме осуществляется в три этапа:

  • Сначала в коде необходимо расставить точки останова щелчком правой кнопкой мыши по коду и выбором пункта раскрывающегося меню "Переключить точку останова".
  • Затем выберите пункт меню Выполнить -> Отладка. Проект автоматически компилируется и запускается.
  • Выполнение программы приостанавливается на первой точке останова. В меню выполнения доступен ряд функциональных возможностей: Выполнить -> Перешагнуть, Выполнить -> Выполнить до курсора и т.д.

В окнах, доступных из меню Окно -> Отладка предоставляются разнообразные сведения о переменных, потоках и т.д.

  • На следующем снимке экрана показаны три важных элемента.
  • точки останова, показанные красным цветом;
  • зеленая линия, показывающая позицию, на которой остановлено выполнение кода;

Классы тестов (инфраструктура Junit)

Важность тестирования повышается по мере роста проекта. Невозможно представить себе тесты в функции main(). Предпочтительно организовать тесты в классы тестов, создавая по классу тестов для каждого класса в приложении. Инфраструктура Junit позволяет смоделировать выполнение приложения.

Создание теста:

  • Установите курсор на класс на вкладке "Проект".
  • Щелкните правой кнопкой мыши класс ---> выберите меню Сервис ---> Тесты Junit ---> Создать тест и подтвердите окно с параметрами настройки по умолчанию.
  • Будет создан класс в пакете "Тесты". Этот класс расширяет класс TestCase — класс инфраструктуры Junit, позволяющий создавать тесты.

Этот класс содержит конструктор, функцию setUp (используется для инициализации тестов), tearDown (используется при закрытии всех тестов) и, наконец, неограниченное количество методов с именами вида testXXX, где XXX — имя теста.

Интернационализация

NetBeans позволяет использовать функции интернационализации (i18n) Java.

Создайте новый стандартный проект с новым классом и введите следующую строку в метод main():

  • System.out.println();
затем разместите курсор между скобками. Выберите пункт главного меню:
  • Сервис ---> Интернационализация ---> Вставка интернационализируемой строки...
Создайте "файл ресурсов" с ключом и соответствующим значением. Нажмите кнопку "ОК" для создания ссылки на ключ.

В строку автоматически вставляется значение:

  • System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n").getString("key_name"));
Вставим новое значение
  • На вкладке "Проект" выберите файл ресурсов, щелкните его правой кнопкой мыши и выберите пункт раскрывающегося меню "Добавить локаль", а затем выберите новое значение.

Новые ссылки добавляются под файлом i18n.properties.

Для выбора значений ключей можно использовать итоговые значения в файле java.util.Locale, например Locale.FRANCE, Locale.CANADA_FRENCH, Locale.CHINESE и т.д. Значение локали необходимо указать во втором аргументе метода getBundle():

    System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n",Locale.JAPAN).getString("key_name"));

Использование CVS

Работа в группе позволяет синхронизировать деятельность с коллегами. В NetBeans встроен клиент CVS. Рассмотрим подключение к серверу CVS проекта NetBeans.

Воспользуемся клиентом CVS:

  • Управление версиями ---> Диспетчер управления версиями
  • Нажмите кнопку "Добавить" для добавления нового рабочего каталога.
  • Введите следующие данные:
    • Рабочий каталог: каталог в операционной системе, куда отправляются исходные файлы.
    • Тип сервера CVS: pserver
    • Имя сервера CVS: cvs.netbeans.org
    • Порт: 2401
    • Имя пользователя: anoncvs
    • Репозиторий: /cvs

  • Должен быть установлен флажок "Выполнить получение".
  • Нажмите кнопку "Далее" в нижней части окна и, наконец, кнопку "Готово".

Выполняется загрузка всех исходных файлов. Получение всех исходных файлов занимает значительное время, однако его можно отменить, ведь это просто тест.

Соединение с базой данных

В большинстве веб-приложений используется база данных, позволяющая хранить данные. В NetBeans встроено средство подключения к базе данных.

Протестируем этот компонент с помощью MySQL. Сначала необходимо загрузить драйверы с этого веб-сайта:
http://dev.mysql.com/downloads/connector/j/

  • Перейдите на вкладку "Среда выполнения", подменю "База данных".
  • Щелкните правой кнопкой мыши Драйверы, Добавить драйверы и выберите драйвер MySQL (файл jar).
  • Выберите драйвер в подкаталоге Driver и создайте подключение: щелчок правой кнопкой мыши ---> Соединиться с помощью... В порядке теста введите следующие данные:
    • Адрес базы данных: jdbc:mysql://localhost:3306/mysql (localhost — локальный компьютер, на котором запущен демон mysql).
    • Имя пользователя и пароль: задаются пользователем.

Далее можно выбирать данные, создавать таблицы и представления (однако эта возможность еще не реализована в самой MySQL).

Автоматическое комментирование

В NetBeans входит графический редактор, позволяющий создавать комментарии к классам Java. Чтобы воспользоваться этим редактором, щелкните правой кнопкой мыши класс Java на вкладке "Проект" или "Файл", а затем выберите пункт меню Сервис -> Автоматическое комментирование.

Откроется интерфейс, в котором показаны все комментируемые методы, классы и т.д. Интерфейс чрезвычайно удобен и более привлекателен, чем написание комментариев вручную.
