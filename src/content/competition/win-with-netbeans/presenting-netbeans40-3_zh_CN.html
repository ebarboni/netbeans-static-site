J2ME

Java 应用程序可以在移动电话、互动电视上使用；Java 是这些设备的理想之选。Sun 提供了 J2ME 开发工具包。通过安装 Mobility 工具包，可以将该开发工具包集成到 NetBeans 中。

可以从以下网站中获取该工具包：https://netbeans.org/kb/trails/mobility.html

然后，您可以获取 J2ME 模板、库和仿真器：

创建 J2ME 应用程序

我们将通过一个模板项目来了解 Mobility 插件的各种功能：

选择 "File"（文件）-> "New Project (Ctrl-Shift-N)"（新建项目 (Ctrl-Shift-N)）。
选择 "Mobile" -> " Mobile Application"（Mobile 应用程序）子类别。
单击 "Next"（下一步），然后将项目命名为 "MyJ2ME"。
将所有选项设置为缺省设置，然后单击 "Finish"（完成）。

将会看到 NetBeans 使用一个包含以下元素的模板类：

  • startApp() 方法
  • pauseApp() 方法
  • destroyApp() 方法
  • commandAction(Command c, Displayable s) 方法
  • 该类扩展了 MIDlet 类，并实现了 CommandListener 接口。

您可以运行项目，将在移动电话仿真器中启动该项目。

项目配置

可以创建不同的配置，以便在不同的上下文中使用我们的项目。

  • 在 "Project"（项目）标签中选择项目，右键单击，然后选择 "Properties"（属性）。


配置窗口

  • 在窗口顶部的组合框中，选择 " Add configuration ... "（添加配置...）。将新配置命名为 "WithKeyboard"。
  • 取消选中 "Use values from Default Configuration"（使用‘缺省配置’中的值）复选框。
  • 在 "Device"（设备）组合框中，选择 "QwertyDevice"，然后单击 "OK"（确定）。
现在，将使用 "WithKeyboard" 配置启动我们的项目：
  • 右键单击该项目，然后选择 "Set as main project"（设置为主项目）以确保该项目是主项目。
  • 在工具栏中，必须选择我们的配置，如下所示：


配置窗口

  • 最后，我们必须选择 "Run"（运行）-> "Run main project ..."（运行主项目...）以执行项目。
  • 您将会看到所选的仿真器（名为 QwertyDevice）正在执行 j2me 软件。
此外，还可以在“配置”中选择混淆级别、Jar 文件的创建方式、Javadoc 的创建方式，等等。

具有 Sun Application Server 的 J2EE

下一个 NetBeans 版本将提供一个关键功能，即，与 Sun Application Server 8.1 集成在一起。此外，还会添加 EJB 和 Web 服务支持。目前，这些功能是为付费 IDE 保留的。

NetBeans Profiler

此插件的目的是提供一个功能强大且非常灵活的性能分析解决方案，该解决方案与 IDE 工作流程紧密集成在一起。通过使用该插件，您可以在执行 Java 软件的同时，随时调整要实施的各种设想。

该插件将在 2005 年 4 月发布，您已经可以在测试版中使用某些功能。

如果要测试该插件，请执行以下操作：

  • 在 http://profiler.netbeans.org/index.html 中下载模块。
  • 在解压缩后，您必须在 "Tools"（工具）-> "Update Center"（更新中心）-> 手动安装模块菜单中选择 .nbm 文件。
  • 在安装后，转到 "Project"（项目）标签，您必须选择包含 main() 函数的项目类，最后在菜单中选择 "Profile"（分析）-> "Profile file..."（分析文件...）。

注意：
在测试版中，您必须使用 1.4 Java 虚拟机。因此，如果您使用 1.5 Java 虚拟机，则可以在编译器中选择 "-source 1.4" 参数或安装 1.4 Java 虚拟机。仅在我的计算机上完成了第二项。

其他

NetBeans 提供了各种不同的“插件”以支持其他语言，如 C/C++、Fortran、甚至 UML，等等。它还发布了用于测试应用程序性能的 Profiler。我们不在此处讨论这些插件，您可以自己试一试。:-)虽然可以在 NetBeans 3.6 版中使用大多数插件，但不能在第四版中使用它们。我们希望尽快解决这一问题。

插件列表：http://plugins.netbeans.org/

其他

优点：

  • 一个包含很多内置功能的大组件面板：XML、文档、数据库、Tomcat，等等。
  • 使人们更愿意使用 Java 编写桌面、Web 和 Mobile 应用程序。

缺点：

  • 在 2005 年春季之前，不会在 4.1 版中发布 J2EE 模块。
  • 无法在第四版中使用大多数“插件”。

小结

由于 NetBeans 完成了项目的最困难部分和辅助部分，它大大减少了程序员的开发工作量。通过使用 NetBeans，您可以节省很多时间和金钱，因而将您从繁重的工作中解脱出来，您可以集中精力完成代码编写工作。
