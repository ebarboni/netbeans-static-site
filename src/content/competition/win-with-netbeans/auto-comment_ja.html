NetBeans 4.0 のクールな自動コメントツール - これでドキュメントの作成がおもしろくなる

Win With NetBeans コンペティションの一部として Jason Stapley 氏から寄稿

はじめに

タイトルで笑わせてしまってすみません。そうしないと、ドキュメントの作成に関する記事を読んでいただけません。
筆者は Sun がリリースした JDK に含まれる JavaDoc ツールをいつも楽しんできましたが、それは提供されたコードの将来の開発者/保守管理者を正しい方向に向けるためのよい方法です。あるいは、皆さんが筆者のようであれば、数か月/数週間/数日後に自分のコードがどのように機能するかを忘れたときに自分自身を正しい方法に向けるためにもよい方法です。

自動コメントツールの機能

それはドキュメントの作成は行いません。それは引き続き皆さんが行う必要があります。ただし、そのツールは JavaDoc フレンドリな方法でコメントを書式化/構築します。
また、これが重要なのですが、それはコメントが見つからない場合に警告します。これは Java のコメントを検証する、より自然な方法です。NetBeans の使用を開始する前、または自動コメントツールを発見する前は、JavaDoc ツールを実行し、受け取った警告に基づいてコメントまたはパラメータを変更していました。この方法は、時間のかかる、うんざりするもので、間違える可能性が高くなったことを意味します。自動コメントツールを使用すると、誤りや欠落がすぐに知らされ、そのときに誤りを訂正できるので、常に訂正するコードまで戻る必要があるわけではありません。

自動コメントツールの使用方法

  1. NetBeans プロジェクトを開きます
  2. Java ファイルを開くか、または「プロジェクト」ウィンドウで Java ファイルまたはクラスノードを選択します。そうしない場合は、自動コメントツールが使用できません。自動コメントツールの使用はすべて、コンテキストに基づいています。別のクラスで自動コメントツールを実行しても、自動コメントの別のインスタンスは開かれません。これは、一度に 1 つの Java ファイルのコメントしか編集できないことを意味します。1 つのプロジェクト上で自動コメントを全体として実行することはできません (機能のヒント)
  3. 「ツール」メニューから、「自動コメント…」を選択します

自動コメントツールが開くと、左側の区画にそのクラス内のメソッドの一覧と、それぞれにアイコンが付いた一連のボタンが表示されます。

これらのボタンはフィルタです。1 つのクラスに多数のメソッドや変数が含まれている場合、自動コメントフィルタはコメントが必要なものだけに焦点を絞るのに役立ちます。

自動コメントフィルタの使用

左側にある最初のボタンをクリックします。このボタンは、白いチェック印の付いた緑のボックスのように表示されます。それはトグルボタンです。このボタンを押すと、JavaDoc 仕様を満たすすべてのコメントが表示されます。

フィルタボタンの詳しい説明が必要な場合。「ヘルプ」メニューに移動し、「ヘルプ」コンテンツをクリックします。次に、検索タブをクリックし、キーワード「自動コメント」を入力します。あるいは、自動コメントツールのメソッドの一覧の横にあるヘルプボタンをクリックするだけです。

フィルタボタンの下にあるクラス一覧の各メソッドの横に表示されるアイコンに注意してください。

「Java コメントは OK」アイコン

白いチェック印の付いた緑のボックスのように表示されるアイコンは、そのメソッドまたは変数のコメントが JavaDoc 仕様を満たしており、ほかに実行する作業がないことを意味します。メソッドまたは変数についての関連した/正確な/最新のコメントが入力されていることのチェックとは別です。

「コメントが見つからない」アイコン

白い X 印の付いた赤のボックスのようなアイコンが表示される場合、これはメソッドのコメントが JavaDoc 仕様を満たしていないか、またはそれに関連づけられたコメントがないことを意味します。JavaDoc の使用を最初に開始したときに、このようなコメントブロックを記述することがたまにあります。 

/*
* Java rocks
*/

当然ながら、追加のアスタリスクを抜かしたため Java ドキュメントには何も表示されませんでした。自動コメントを通じて「JavaDoc コメントテキスト:」テキストフィールドにコメントを入力した場合、自動コメントは適切な書式設定を使ってそのコメントをコードに書き込みます。例:

/**
* Java rocks

*/

エラーが「コメントが見つからない」である場合。自動コメントによって、メソッド一覧の下にあるリストボックスで警告されます。その下のエラーメッセージは「JavaDoc comment is missing (JavaDoc コメントが見つかりません)」になります。すでにメソッドまたは変数にコメントしているのに、このエラーが警告されている場合は、ソースの表示ボタンをクリックすることで、その追加の * が抜けていないことを確認できます。同じことの繰り返しになりますが、自動コメントツールでは、そのすべてがコンテキストに基づいています。メソッドが選択されていない場合は、ソースの表示ボタンが灰色になります。

「部分エラーまたは完全に間違っている」アイコン

もっとも関心の高いアイコンは部分エラーアイコンであり、これは稲妻が貫通した三角形のように表示されます。ほとんどの場合、パラメータが含まれるメソッドコメントに @param タグを含めないときにこのエラーが表示されます。自動コメントは、タグが見つからないパラメータを通知します。例:The tag for the parameter <パラメータ名> is missing. (パラメータ <パラメータ名> のタグが見つかりません。)

部分エラーを受け取るもう 1 つの例は、不要なタグをコメントに追加する場合です。たとえば、クラスではなくクラスメソッドに author タグを追加する場合です。次の例を見てください。 


/**
 * This method does something
 * @param Something
 * @Author Jason Stapley – This is an error. Not me, the Author tag!
 */
Void doSomething(String Something) {
    System.out.println(Something);
}

この種のエラーを自動コメントで訂正するには、タグ一覧で不要なタグを選択し、削除ボタンをクリックします。この続きは、次のタグを追加する見出しのところで説明します。

コメントブロックへのタグの追加

自動コメントツールのタグ一覧の下にある「新規... 」ボタンをクリックすると、新しいタグウィンドウが表示されます。ここには、コメントブロックに追加できるタグの種類が一覧表示されます。パラメータフィールドに名前を指定する必要があるパラメータの場合、もっとも一般的なタグは、必要なタグの種類を選択したあとの @Param と @Return です。このパラメータはメソッド内のパラメータと一致する必要があります。間違えても心配しないでください。自動コメントによって警告されます。

次に、「説明」フィールドにコメントを追加します。すべてのパラメータと return 文にコメントが付けられるまで繰り返します。別のメソッドをクリックするか、再表示をクリックします。コメントを付けたばかりのメソッドが「コメントは OK」アイコンに変わるはずです。そうでない場合は、メソッド一覧の下にあるエラーメッセージのテキストボックスで、問題が発生している原因を確認します。

タグの順序

JavaDoc が適切であるように定義された順序でタグを置く必要はありません。それは、皆さん自身にとって論理的なものでも、皆さんのコードを読むほかの人にとって論理的なものでもかまいません。自動コメントを使用すると、コメントブロック内のタグの順序を変えることができます。たとえば、return タグが常にリストの最後のタグになるようにする場合は、それを選択し、「下へ移動」ボタンをクリックしてリストの一番下に移動します。それがリストの先頭になるようにする場合は、それを選択し、「上へ移動」ボタンを再度クリックします。これらのボタンは、実行している処理のコンテキスト内でのみ使用できます。

HTML タグ

JavaDoc では HTML を作成するため、HTML タグをコメントに追加できます。コメントを付けるメソッドまたはフィールドを選択し、「JavaDoc Comment Text:」フィールドに「<」と入力して数秒待ちます。使用できるタグの一覧が表示されるはずです。コードを編集する場合と同様に、オブジェクト名のあとにドットを入力して、メソッドの一覧を表示できます。これはタグの場合も同じです。タグの「説明」フィールドにもオートコンプリート機能があります。

また、JavaDoc でもっともよく使われる各種タグが「自動コメント」画面の右下にあります。これらは <B> <I> <U> <CODE> <PRE> <@link> です。

非常に強力です。

自動コメントの再表示

自分のコードに戻る必要があることがたまにあります。たとえば、コメントブロックを 2 つのアスタリスクで始めるのを忘れた場合、自動コメントはそのコメントを見つけることができないため、「ソースを表示」をクリックして確認する必要があります。間違いをして、自動コメントに戻る場合は、常に「再表示」ボタンをクリックして同じ作業を繰り返さないようにすることが賢明です。

自動修正

かりに多数のパラメータを含むメソッドがあるとしたら、すべてのパラメータ名を入力するのにある程度の時間がかかります。自動修正機能は、皆さんに代わってパラメータのすべてのタグを作成し、それらに名前を付けるので、皆さんはコメントを追加するだけで済みます。これを行うには、コメントを付けるメソッドを選択し、「自動修正」ボタンをクリックしてから、パラメータの「説明」フィールドにコメントを追加します。

終了するとき

ツールバーの「すべてを保存」ボタンをクリックして、行なった作業を必ず保存してください。

まとめ

これで自動コメント機能の使用方法とそれによってどれだけの時間を節約できるかをおわかりいただけたと思います。コードにコメントを付けないことは許されません。1 つの実験として、古いコードの一部でそれを実行してください。驚くことになります。
