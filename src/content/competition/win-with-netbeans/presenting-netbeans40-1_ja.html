はじめに:

統合開発環境とは、プログラマを支援するツールを集めたものです。NetBeans はそれらの 1 つであり、Java に特化しています。こうして、心を解放し、自身のコードに容易に集中できます。

このソフトウェアのほとんどの機能について解説しますが、もっとも重要なものをまず説明し、その後、チュートリアルの実習を行います。最後に、プラグインのプレゼンテーション、およびこの IDE の利点と欠点で締めくくります。

これは Sun Microsystems によって開発されており、その公式 Web サイト www.netbeans.org から無料でダウンロードできます。

警告: この記事の目的は、すべての機能を紹介することではなく、その可能性のオーバーヘッドビジョンを提示することです。未解決の問題がある場合は、公式 Web サイトやドキュメントを参照することをお勧めします。

歴史:

1996: Xelfi という名前のチェコ企業によって NetBeans プロジェクトが開始されました。彼らの目的は、Delphi に似た Java 向け IDE を Java で記述することでした。「Developer」 2.0 と 2.1 という、2 つの商用版が存在しました。

1999 年 5 月: 3.0 Beta 版がリリースされます。

1999 年 10 月: Sun Microsystems が NetBeans を買収します。この IDE の名前が「Sun Forte For Java Community Edition」に変更されます。このバージョンは無料ですが、プロフェッショナル版は購入する必要があります。

2000 年 6 月: Sun が NetBeans をオープンソースとしてリリースします (名前を元の名前に変更)。

2004 年 12 月: Java 5 のサポート、改善されたプロジェクトマネージャー、改善されたグラフィカルインタフェース、プロジェクトへの Ant の導入など、新しい機能を含むバージョン 4 がリリースされます。

今後について:

2005 年春:バージョン 4.1 が J2EE モジュールとともにリリースされます。

Sun のポリシーでは、NetBeans は、ONE (One Net Environment) の IDE である「Sun ONE Studio」のルートソフトウェアです。ONE は、ネットワークや開発の専門家を支援すると思われる一連のアプリケーションを提供することを期待されたプロジェクトです。このプロジェクトには、「Sun ONE Application Server」や「Sun ONE Web Services Platform」などが含まれています。

NetBeans は Win32、Linux、Mac OS X、および Solaris 上で動作します。そのアーキテクチャーは 100% Java で記述されています。

さらに、ツールはすべて無料です。NetBeans は、プログラマのスイスナイフです。

全般的なプレゼンテーション:


プロジェクトマネージャー:
「ファイル」->「新規プロジェクト」を選択しましょう

NetBeans は、すぐに使用できるプロジェクトテンプレートを提案します。

  • 一般アプリケーション用 (コマンド行ソフトウェアやグラフィカルインタフェースなど)
  • Web サイト用。4.1 バージョンでは、EJB や Web サービスといった J2EE のためのテンプレートが提供されます。
  • コード例を含む「サンプル」項目メニュー
テンプレートプロジェクトを使うと作業が簡単になりますが、それは、Web ソフトウェア (後述する Tomcat) など、環境内のソフトウェアを NetBeans が自動的に構成してくれるからです。

「一般」->「Java アプリケーション」を選択し、「NetBeans」という名前のプロジェクトを作成します。

2 つのパッケージがあります。

「ソースパッケージ」には、ユーザーのソースコードが含まれます。そこにはファイルや画像などを配置します。

「テストパッケージ」には、JUnit ファイルのソースコードが含まれます。JUnit はテスト用のフレームワークです (これについては後述します)。

警告: このパッケージは、Java 言語のパッケージと同じものではありません。

ソフトウェアに新しい要素を追加する場合は、プロジェクトタブでプロジェクトを選択し、メニューで「ファイル」->「新規ファイル...」を選択します

このウィンドウには、Java パッケージからサーブレット、Swing グラフィカルインタフェースに至るまで、多数のファイルテンプレートが含まれています。

ここでは Java クラスを作成します。

「Java クラス」->「Java クラス」

NetBeans の機能を理解するために、これら多くの種類のテンプレートをすべてテストすることをお勧めします。

ところで筆者は、2 つの Java パッケージ netbeans および netbeans.test と、2 つの Java クラス Main.java および classeTest.java を作成しました

テンプレートファイルを作成する場合は、プロジェクトのいずれかのファイルを右クリックし、「テンプレートとして保存...」を選択します

プロジェクトをコンパイル、実行、および配備する方法については、後述します。

サポートされているファイル:

IDE は、一般的に使用されるファイルをサポートすることでプログラマの作業を支援する必要があります。NetBeans は、次の各ファイルの構造を認識し、一部の字句や構文に色を付けることができます。

  • Java ファイル: 字句や構文の色分け、自動補完、および誤り修正あり
  • XML: 構文の色分け、DTD および XSL ジェネレータ、およびバリデータあり。
  • HTML および CSS: 構文の色分けあり。

自動補完:

自動補完は、ほかの一部の IDE では「インテリセンス」と呼ばれることもありますが、これを使用すると、単語の末尾の提案が可能になります。これを理解するには例を挙げるのが得策です。

ここでは新しい行を開始し、「String」と記述して Ctrl-Space キーを押すと、使用可能なクラスやインタフェースの名前を含むポップアップメニューが開き、さらにドキュメントまで表示されます。

たった今確認したように、NetBeans には Java ドキュメントが統合されています。

自動修正:

次の行を記述した場合、
StringBuffer sb = new StringBuffer("test",5);

NetBeans はこれにアンダースコアを付けます。その理由は、NetBeans が表示しているように、この 2 つの引数を持つ使用可能なコンストラクタが存在しないからです。

自動修正は、ファイルをコンパイルする前でも使用できます。

リファクタリング:

リファクタリングとは、コードを 1 行も記述/変更することなしにパッケージやファイルの名前変更を可能にする機能のことです。たとえば、あるパッケージの名前を変更する必要がある場合は、それを「プロジェクト」タブで選択し、メニューで「リファクタリング」->「名前を変更...」を選択します。

「次へ」をクリックすると、NetBeans によって実行されるすべてのアクションの簡易説明が表示されます。

「リファクタリングを実行」をクリックすると、「netbeans」パッケージの名前が「supinfo.labo.sun」に変更され、main.java ファイルがその新しいパッケージに移動されます。

ファイルの構成:

各ディレクトリにはそれぞれ異なるファイルが含まれます。

  • build にはクラスファイルが含まれます。
  • dist には、プロジェクトの最終的なファイルが含まれます。アプリケーションの JAR とJavadoc が存在します。
  • nbproject には、プロジェクトのコンパイル、実行、配備などの方法を決定する Ant 構成ファイルが含まれます。
  • src にはプロジェクトのソースが含まれます
  • test にはプロジェクトのテストファイルが含まれます (JUnit については後述します)
  • その他のファイル: コンパイルに使用される build.xml (これは Ant ファイルの 1 つ) と、プロジェクトを定義する manifest.mf

ドキュメントファイルを生成する:

ドキュメントを作成すると、コードのアーキテクチャーをほかのプログラマが理解できます。JDK で提供されている javadoc ツールを、NetBeans のグラフィカルインタフェース経由で使用できます。

javadoc は、(/** @autor name_of_the_autor */ などのシンボルを含む) コードのコメントを使用して、ドキュメントファイルを出力します。

メニューで「構築」->「プロジェクトの Javadoc を生成」を選択すると、ドキュメント化が行われます。プロジェクトの dist/javadoc 内のファイルでこれを参照できます。

自動コンパイル:

コンパイルを担うのは XML ファイルですが、これらのファイルは Ant によって解釈されます。特に何もしなくとも、この部分は自動的に生成されます。

コンパイルは、「実行」--->「主プロジェクトを実行」を行ってソフトウェアを実行する際に、自動的に行われます。コンパイルを手動で起動するには、メニューで「構築」->「主プロジェクトを構築」を選択します。アプリケーションの JAR ファイルは、アプリケーションのルートディレクトリの「dist」ディレクトリ (「ファイル」タブで表示可能) に格納されます。

Ant によって使用される XML ファイルは、アプリケーションのルートディレクトリ内の「nbproject」ファイルに自動的に生成されます。これらはあとで、コンパイル時、ドキュメント生成時、起動時に使用されます。

Ant の動作方法をご存じない方には、その公式ドキュメントに目を通すことをお勧めします。

「デバッグ時」

コードを記述している間は、ソフトウェアの実行の各部分を制御することが重要です。アプリケーションをステップ単位で起動する必要がある場合は、3 つの段階が存在します。

  • ソースコード上で右クリックして「ブレークポイントを切り替え」を行うことで、コード内にブレークポイントを設定する必要があります
  • メニューで「実行」->「デバッグ」を選択すると、プロジェクトが自動的にコンパイルされ、起動されます。
  • 最初に検出されたブレークポイントでプログラムが一時停止されます。実行メニューには、「実行」->「ステップオーバー」、「実行」->「カーソルまで実行」など、多くの機能が用意されています。

「ウィンドウ」->「デバッグ」の各ウィンドウは、変数やスレッドなどに関する多くの情報を提供します。

  • 次のスクリーンショットには、重要なものが 3 つ含まれています。
  • 赤色で表示されたブレークポイント
  • ソフトウェアの停止位置を示す緑色の行

テストクラス (JUnit フレームワーク)

プロジェクトの規模が大きくなるにつれ、テストの重要性がますます高まります。それらを main() 関数内に記述することは考えられません。それらをテストクラスとして構成し、アプリケーション内のクラスごとにテストクラスを 1 つずつ作成したほうが得策です。したがって、JUnit フレームワークは、アプリケーション実行のシミュレーションを可能にします。

テストの作成方法:

  • 「プロジェクト」タブで、クラス上にカーソルを設定します。
  • 右クリックして「ツール」--->「JUnit テスト」--->「テストを作成」を選択し、デフォルトのオプションとともに表示されるウィンドウを確認します。
  • 「テストパッケージ」内にクラスが作成されます。これは TestCase を拡張したものですが、この JUnit フレームワークのクラスを使用すると、テストを作成できます。

このクラスには、コンストラクタ、(テスト初期化子として使用される) setUp という名前の関数、(すべてのテストの終了時に使用される) tearDown、そして最後に、testXXX (XXX はテストの名前) という名前を持つ任意の数のメソッドが含まれます。

国際化:

NetBeans では国際化 (i18n) Java 機能を使用できます。

新しいクラスを含む新しい一般プロジェクトを作成し、その main() メソッド内に次の行を記述します。

  • System.out.println();
そして、カーソルを括弧の間に設定します。メニューで次を選択します。
  • 「ツール」--->「国際化」--->「国際化文字列の挿入...」
キーと関連する値を含む「バンドル」を作成します。「了解」を押すと、キーへの参照が作成されます。

NetBeans によって行に値が挿入されます。

  • System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n").getString("key_name"));
新しい値を挿入します。
  • 「プロジェクト」タブでバンドルファイルを選択し、右クリックして「ロケールを追加」を選択し、新しい値を選択します。

i18n.properties ファイルの下に新しい参照が追加されます。

キーの値を選択する場合は、Locale.FRANCE や Locale.CANADA_FRENCH、Locale.CHINESE などの、java.util.Locale パッケージの final 値を使用できます。getBundle() メソッドの第 2 引数にロケール値を設定する必要があります。

    System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n",Locale.JAPAN).getString("key_name"));

CVS の使用

チーム作業では、自分の作業をほかのメンバーと同期させることができます。NetBeans には組み込み CVS クライアントが用意されています。NetBeans プロジェクトの CVS サーバーに接続します。

CVS クライアントを使用してみましょう。

  • 「バージョン管理」--->「バージョン管理マネージャー...」
  • 「追加」をクリックして新しい作業用ディレクトリを追加します。
  • 次の情報を入力します。
    • 作業用ディレクトリ: ソースの送信先にする必要がある、オペレーティングシステム内のディレクトリ。
    • CVS サーバーの種類: pserver
    • CVS サーバー名: cvs.netbeans.org
    • ポート: 2401
    • ユーザー名: anoncvs
    • リポジトリ: /cvs

  • 「チェックアウトの実行」をオンにする必要があります
  • ウィンドウの最下部にある「次へ」をクリックし、最後に「完了」をクリックします。

NetBeans によって、すべてのソースファイルがダウンロードされます。すべてのソースファイルのチェックアウトには長い時間がかかりますが、これはテストにすぎないので取り消すことができます。

データベースへの接続

ほとんどの Web ソフトウェアは、データベースを操作することで情報を保存できるようにします。NetBeans には、データベースに接続するための組み込みツールがあります。

MySQL を使用してこの部分をテストします。まず、次の Web サイトでドライバをダウンロードする必要があります。
http://dev.mysql.com/downloads/connector/j/

  • 「実行時」タブを選択し、サブメニュー「データベース」を選択します。
  • 「ドライバ」を右クリックして「ドライバを追加」を選択し、MySQL ドライバ (これは JAR ファイルである) を選択します。
  • 「ドライバ」サブディレクトリでドライバを選択し、右クリックして「接続...」を選択して接続を作成します。このテストでは次の情報を入力しました。
    • データベース URL: jdbc:mysql://localhost:3306/mysql (ここで localhost は、mysql デーモンが実行されているローカルマシン)。
    • ユーザー名とパスワード: これは皆さんの情報です。

データの選択、表の作成、ビューの作成を行うことができます (ただし、この機能はまだ MySQLに存在しない)。

自動コメント:

Java クラスのコメントの記述を支援するグラフィカルエディタがあります。これを使用するには、「プロジェクト」または「ファイル」タブで Java クラスを右クリックし、「ツール」->「自動コメント...」を選択します

インタフェースに、コメントを追加できるすべてのメソッドやクラスなどが表示されます。これは非常に簡単であり、コメントを手動で記述するよりも魅力的です。
