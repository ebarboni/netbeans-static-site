J2ME

Os aplicativos Java podem funcionar em celulares, televisões interativas e são um eldorado para o Java. A Sun fornece um kit de desenvolvimento para o J2ME. Este é integrado no NetBeans, ao instalar o kit de mobilidade.

Você o obtém neste endereço da Web: https://netbeans.org/kb/trails/mobility.html

Então, receberá os modelos, bibliotecas e emuladores J2ME:

Criar uma aplicativo J2ME

Vamos usar um projeto modelo para explorar as diferentes funcionalidade do plug-in de mobilidade:

Selecione Arquivo > Novo projeto (Ctrl-Shift-N).
Escolha a subcategoria "Móvel" -> "Aplicativo móvel".
Clique em Próximo e nomeie o projeto como "MyJ2ME".
Ao deixar todas as escolha de parêmetros como padrão, clique em "Terminar".

Você verá que o NetBeans usou uma classe de modelo com estes elementos:

  • o método startApp()
  • o método pauseApp()
  • o método destroyApp()
  • o método commandAction(Command c, Displayable s)
  • a classe estende a classe MIDlet e implementa a interface CommandListener.

Você pode executar o projeto que será iniciado em um emulador de telefone celular.

Configurações de projeto

Nós podemos criar diferentes configurações, nos permitindo usar nosso projeto em diferentes contextos.

  • Selecione o projeto na aba Projeto, clique com o botão direito do mouse e selecione "Propriedades".


Janela Configuração

  • No topo da janela, na caixa combo, selecione "Adicionar configuração...". Nomeie a nova configuração como "WithKeyboard".
  • Cancele a seleção da ciaxa de seleção "Usar valores da configuração padrão".
  • Na caixa combo "Dispositivo", selecione "Dispositivo Qwerty" e clique em OK.
Agora, vamos iniciar seu projeto com a configuração "WithKeyboard":
  • Clique com o botão direito do mouse no projeto e selecione "Definir como projeto principal", para assegurar que este seja o projeto principal.
  • Na barra de ferramentas nós temos que selecionar nossa configuração, como abaixo mostrado:


Janela Configuração

  • Finalmente, nós temos que executar o projeto ao selecionar "Executar -> Executar projeto principal...".
  • Você verá o emulador selecionado, denominado QwertyDevice, executando seu software j2me.
Além disso, as configurações permitem selecionar o nível de ofuscação, como o arquivo jar é criado e como o javadoc é criado,...

J2EE com o Sun Application Server

A proxima versão do NetBeans irá fornecer um recurso chave, que é a integração com o Sun Application Server 8.1. Além disso, o suporte ao EJB e Serviços da Web será adicionado. Estas funcionalidade ainda estão reservadas para IDEs dispendiosos.

Perfilador do NetBeans

O objetivo deste plug-in é o de fornecer uma solução poderosa e flexível para perfilamento e que está bem integrada no fluxo de trabalho do IDE. Ai utilizá-lo, você terá uma "visão do overhead" de seu software Java em tempo de execução.

Ele será liberado em Abril de 2005, e alguma funcionalidade já pode ser usada na versão beta.

Caso deseje testá-lo:

  • Faça o download do módulo em http://profiler.netbeans.org/index.html.
  • Após extraído, você precisa selecionar o arquivo. nbm no menu "Ferramentas -> Centro de atualização -> Instalação manual dos módulos ".
  • Após instalado, siga para a aba Projeto, e selecione classe de seu projeto que contém a função main() e, finalmente, selecione o menu Perfilador -> Arquivo do perfilador ...

Aviso:
Na versão beta, é preciso usar a máquina virtual Java 1.4. Consequentemente, se você tiver a 1.5, poderá selecionar o argumento "-source 1.4" com o compilador ou instlar a 1.4. Somente o segundo item completado em meu computador.

Diversos

O NetBeans tem uma ampla variedade de "plug-ins" para suportar outras linguagens, como C/C++, Fortran, ... ou até UML. Um perfilador, permitindo testar o desemepenho de nosso aplicativo, também foi liberado. Nós não iremos falar dele aqui e vamos deixar que você o faça:-) Embora, a maioria do plug-ins estão disponíveis para a versão 3.6 do NetBeans, eles não são utilizáveis com a quarta versão. Esperamos que isso seja obtido em breve.

Lista de plug-ins: http://plugins.netbeans.org/

Diversos

Vantagens:

  • Uma grande paleta de funcionalidades incorporadas : xml, documentação, bacos de dados, tomcat ...
  • atrativas para trabalhar com o Java em aplicativos desktop, Web e móveis.

Desvantagens:

  • O módulos J2EE não serão liberados antes da Primavera de 2005, na versão 4.1.
  • A maioria dos plug-ins nãi estã disponíveis para a quarta versão.

Conclusão

O NetBeans permite uma grande facilidade de desenvolvimento para os programadores, ao tomar conta de partes difícies e secundárias de seus projetos. Usando o NetBeans, você irá economizar tempo, dinheiro e paz de espírito. Consequentemente, você pode se concentrar com facilidade em seu código.
