简介：

集成开发环境是一个旨在为程序员提供帮助的工具包。NetBeans 就是其中的一个，它专用于 Java。这样，您就可以腾出时间和精力，将重点放在重新编码上。

我们将介绍该软件的大部分功能（大多数重要功能是首次介绍的），然后我们学习实践教程。最后，我们将演示插件并说明该 IDE 的优缺点。

它是由 Sun Microsystems 开发的，可以在官方网站上免费下载：www.netbeans.org。

警告：本文的目标并不是介绍所有功能，而是说明它可以帮助您实现各种各样的设想。如果您有任何未解决的问题，建议您查阅官方网站和文档。

历史：

1996 年：捷克的一家名为 Xelfi 的公司推出了 NetBeans 项目。他们希望使用 Java 为 Java 编写类似 Delphi 的 IDE。共有两个商业版本："Developer" 2.0 和 2.1。

1999 年 5 月：发布了 3.0 测试版。

1999 年 10 月：Sun Microsystems 购买了 NetBeans。IDE 名称更改为 "Sun Forte For Java Community Edition"。此版本是免费的，而专业版则必须购买。

2000 年 6 月：Sun 以开源形式发布了 NetBeans（将其重命名为原始名称）。

2004 年 12 月：发布了第四个版本，其中包含很多新功能，例如，Java 5 支持、更好的项目管理器、改进的图形界面、在项目中包含 Ant，等等。

将来：

2005 年春季。将发布 4.1 版，其中包含 J2EE 模块。

在 Sun 策略中，NetBeans 是 "Sun ONE Studio" 的根软件，即 ONE 的 IDE (One Net Environment)。ONE 是一个计划，旨在提供一组应用程序以帮助网络和开发专业人员。在该项目中，我们可以看到 "Sun ONE Application server"（Sun ONE 应用服务器）、"Sun ONE Web Services platform"（Sun ONE Web 服务平台），等等。

NetBeans 适用于 Win32、Linux、MacOs X 和 Solaris。体系结构百分之百是使用 Java 编写的。

此外，我们看到的所有工具都是免费的。NetBeans 是程序员的编程利器。

支持的文件：

IDE 必须支持常用的文件才能帮助完成程序员的工作。NetBeans 可以识别以下文件的结构、某些词法单词着色和语法：

  • Java 文件，具有词法和语法着色、自动完成和错误更正。
  • xml，具有语法着色、DTD 和 XSL 生成器以及验证器。
  • HTML 和 css，具有语法着色。

文件的组织结构：

我们可以在不同的目录中看到不同的文件：

  • build 包含类文件。
  • dist 包含项目的最终文件，其中包含应用程序的 JAR 和 Javadoc。
  • nbproject 包含 Ant 配置文件，用于确定如何编译、执行和部署项目等。
  • src 包含项目的源文件。
  • test 包含项目的测试文件（我们稍后介绍 Junit）。
  • 其他文件：用于编译的 build.xml（这是一个 Ant 文件）以及用于定义项目的 manifest.mf。

生成文档文件：

通过创建文档，可以让其他程序员了解您的代码结构。我们可以通过 NetBeans 图形界面使用 JDK 中提供的 Javadoc 工具。

Javadoc 使用代码注释编写文档文件（例如，使用符号 /** @autor name_of_the_autor */）。

通过在菜单中选择 "Build"（生成）-> "Generate JavaDoc For Project"（为项目生成 Javadoc），即可完成文档编写工作。您可以在项目的 dist/javadoc 文件中查阅该文档。

自动编译：

编译是由 Ant 解释的 XML 文件负责执行的。如果没有特殊需要，将自动生成该部分。

编译是通过 "Run"（运行）-> "Run Main Project"（运行主项目），在软件执行时自动完成的。也可以在菜单中选择 "Build"（生成）-> "Build main project"（生成主项目）以手动启动编译。应用程序的 JAR 文件位于应用程序根目录下的 "dist" 目录中，您可以在 "File"（文件）标签中看到该目录。

Ant 预计使用的 XML 文件是在应用程序根目录下的 "nbproject" 文件中自动生成的。以后编译、生成文档和启动时，将使用这些文件。

如果您不知道 Ant 是如何工作的，建议您阅读正式文档。

“调试时”

在编写代码时，一定要控制如何执行软件的每个部分。如果要逐步启动应用程序，则需要三个阶段：

  • 您必须在源代码中右键单击，然后选择 "Toggle Breakpoint"（切换断点）以在代码中设置断点。
  • 在菜单中选择 "Run"（运行）-> "Debug"（调试），将自动编译并启动项目。
  • 程序在遇到的第一个断点处暂停。“运行”菜单提供了很多功能，例如："Run"（运行）-> "Step Over"（步过），"Run"（运行）-> "Run to Cursor"（运行至光标位置），等等。

"Window"（窗口）-> "Debug"（调试）中的窗口提供了一些有关变量、线程等的信息。

测试类（Junit 框架）

当项目越来越大时，测试将变得越来越重要。使用 main() 函数编写这些测试是无法想象的。我们应在测试类中更好地组织这些测试，以便为应用程序中的每个类创建一个测试类。因此，您可以通过 JUnit 框架模拟应用程序的执行情况。

如何创建测试：

  • 在 "Project"（项目）标签中，将光标放在一个类上。
  • 右键单击，选择 "Tools"（工具）-> "Junit Tests"（Junit 测试）-> "Create Test..."（创建测试...），然后验证窗口是否显示了缺省选项。
  • 将在"Test Packages"（测试包）中创建一个类。该类扩展了 TestCase，可通过该 JUnit 框架的类创建测试。

该类包含一个构造函数、一个名为 setUp 的函数（用作测试初始化函数）、tearDown（在所有测试结束时使用）以及不限数量的 testXXX 方法，其中 XXX 是测试的名称。

国际化：

NetBeans 允许您使用国际化 (i18n) Java 函数。

创建一个新的标准项目（包含一个新类），在 main() 方法中编写以下行：

  • System.out.println();
然后将光标放在引号之间。在菜单中选择：
  • "Tools"（工具）-> "Internationalization"（国际化）-> "Insert Internationalized String..."（插入国际化字符串...）。
创建一个“包”，其中包含一个键和相关的值。按 "OK"（确定），将创建对该键的引用。

NetBeans 将在下面的行中插入一个值：

  • System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n").getString("key_name"));
我们将插入一个新值：
  • 在 "Project"（项目）标签中，选择包文件，右键单击，选择 "Add Locale ..."（添加语言环境...），然后选择一个新值。

将在 i18n.properties 文件下面添加新引用。

要选择键值，您可以使用 java.util.Locale 包中的最终值（如 Locale.FRANCE、Locale.CANADA_FRENCH、Locale.CHINESE 等）。您必须将语言环境值放在 getBundle() 方法的第二个参数中：

    System.out.println(java.util.ResourceBundle.getBundle("supinfo/labo/sun/i18n",Locale.JAPAN).getString("key_name"));

使用 CVS

通过团队合作，您可以与同事之间同步您所做的工作。NetBeans 提供了一个内置 CVS 客户端。我们将连接到 NetBeans 项目 CVS 服务器。

让我们使用 CVS 客户端：

  • 选择 "Versioning"（版本控制）-> "Versioning Manager ..."（版本管理器...）
  • 单击 "Add"（添加）以添加新的工作目录。
  • 输入以下信息：
    • Working Directory（工作目录）：必须将源文件发送到的操作系统目录。
    • CVS Server Type（CVS 服务器类型）：pserver
    • CVS server name（CVS 服务器名称）：cvs.netbeans.org
    • Port（端口）：2401
    • Username（用户名）：anoncvs
    • Repository（资源库）：/cvs

  • 必须选中 "Perform Checkout"（执行签出）。
  • 单击窗口底部的 "Next"（下一步），最后单击 "Finish"（完成）。

NetBeans 将下载所有源文件。签出所有源文件需要很长时间，但您可以取消签出，因为这只是一个测试而已。

连接到数据库

大多数 Web 软件都使用数据库来保存信息。NetBeans 具有一个内置工具，可用于连接到数据库。

我们将使用 MySql 测试此部分。首先，您必须从以下网站中下载驱动程序：
http://dev.mysql.com/downloads/connector/j/

  • 选择 "RunTime"（运行环境）标签，然后选择 "DataBase"（数据库）子菜单。
  • 右键单击 "Drivers"（驱动程序），选择 "Add drivers..."（添加驱动程序...），然后选择 "MySQL" 驱动程序（这是一个 JAR 文件）。
  • 在 "Drivers"（驱动程序）子目录中，选择您的驱动程序，然后右键单击并选择 "Connect Using"（连接设置...）以创建连接。对于此测试，我们输入了以下信息：
    • 数据库 Url：jdbc:mysql://localhost:3306/mysql（其中，localhost 是我们的本地计算机，将在其中运行 mysql 守护进程）。
    • 用户名和口令：由您自行确定！

您可以选择数据，创建表以及创建视图（但此功能在 MySQL 中还不存在）。

自动注释：

它提供了一个图形编辑器以帮助您在 Java 类中编写注释。要使用该图形编辑器，您必须在 "Project"（项目）或 "File"（文件）标签中右键单击某个 Java 类，然后选择 "Tools"（工具）-> "Auto-comment..."（自动注释...）。

界面将显示所有可注释的方法、类，等等。这是很方便的，而且比手动编写注释更有吸引力。
