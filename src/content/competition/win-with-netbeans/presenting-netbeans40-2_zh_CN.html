标准应用程序中的 hello world

我们将启动一个简单的标准项目，这是一个输出 "hello word" 的应用程序。接下来，我们为其添加图形界面。

执行以下操作：

选择 "File"（文件）-> "New Project"（新建项目）。
选择 "Standard"（标准）-> "Java Application"（Java 应用程序）。
然后将项目命名为 "tutorial"。

在 main 函数中编写以下内容：

    System.out.println("Hello World");
然后，我们编译并执行该项目：
"Build"（生成）-> "Build main project ..."（生成主项目...）
"Run"（运行）-> "Run main project"（运行主项目）...

输出窗口将输出 "Hello World" 以及 Ant 输出的其他行。


编辑图形界面：

我们稍后将使用图形界面编辑器。它基于 Swing，后者是最常用的 Java 图形库之一。

执行以下操作：
选择 "File"（文件）-> "New Project ..."（新建项目...）。
选择 "Java Gui Forms"（“Java Gui 窗体）-> "Jframe"。
然后将类命名为 "javaGui"。

在 "Palette"（组件面板）标签中，选择一个 JLabel，然后将其放到在屏幕中间绘制的 JFrame 上。在 "Properties"（属性）标签中，使用类似 "Hello World" 的值填充 "text" 变量。建议您耐心等待，Swing 是一个较大的库，可以通过该图形编辑器使用它的大多数功能。

在执行应用程序之前，必须检查我们的 JFrame 类是否为启动时使用的类。在菜单中选择 "File"（文件）-> "tutorial Properties..."（tutorial 属性...）-> "Run"（运行）-> "Running Projects"（运行项目），然后在主类字段中输入 "tutorial.javaGui" 或使用浏览按钮。这样，应用程序将在 tutorial.javaGui 的 main 方法中启动。

启动应用程序，然后查看 "Hello World" 窗口（厉害吧？）：

警告：
NetBeans 图形编辑器使用 Java 未提供的一些图形库。因此，您必须为以后使用 NetBean 图形编辑器生成的软件提供这些库。不过，如果您不了解 Swing API 的基础知识，建议您不要使用该图形界面编辑器。一开始，您最好自己编写图形界面。


Tomcat 集成：

Web 开发是从去年开始兴起的。Java 非常适合此类环境。NetBeans 附带提供了 Tomcat Web 应用服务器以方便进行开发。在此部分中，我们将创建一个 JSP 和一个 Servlet。目标并不是了解 Web 技术，而是了解如何通过 NetBeans 使用 Tomcat。


创建 Web 项目：

执行以下操作：


选择 "File"（文件）-> "New Project"（新建项目）。
选择一个 Web 项目，然后选择 "Web Application"（Web 应用程序）模板。
将项目命名为 TomcatTest。

将创建一个项目，然后将其添加到 Tomcat 应用服务器中。

编辑 JavaBean：

可以使用其他方法来处理窗体的数据。JavaBean 就是其中的一种方法，它包含使用 Java 类收集值的功能。该类必须应用设计模式（针对每个值使用 getter 和 setter）。

如果要执行 JavaBean：

  • 将光标放在 Web 项目上。
  • 选择 "File"（文件）-> "New File..."（新建文件...）。
  • 选择 "Java Classes"（Java 类）-> "Java Classe"（Java 类）。
  • 将 Java 类命名为 "bean"，将它的包命名为 "beanPackage"。

编辑 Servlet：

Servlet 是用于创建 Web 页的 Java 类。事实上，将在 Servlet 中自动转换 JSP 页以进行编译。

我们将使用 Servlet 创建一个窗体。

  • 如果要创建 Servlet：
  • 选择 "File"（文件）-> "New File..."（新建文件...）。
  • 选择 "Web" 菜单 -> "Servlet"。
  • 将 Servlet 命名为 "destination_servlet"，将包命名为 "servletPackage"。

destination_servlet 应如下所示：

destination_servlet.java

如果我们打开 WEB-INF/web.xml，则可以看到已配置且可直接使用的 Servlet。

我们仍使用 JSP 窗体：

在“项目”标签中右键单击该项目，然后选择“运行项目”。将打开 Web 浏览器以显示 index.jsp 文件。您将单击第三个链接以测试基于 Servlet 的解决方案。

在 "Runtime"（运行环境）标签中，选择“服务器注册”子目录，您可以直接访问某些 Tomcat 功能。
