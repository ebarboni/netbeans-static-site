一般アプリケーションでの Hello World

単純な一般プロジェクト (「Hello World」を出力するアプリケーション) を開始します。次に、それにグラフィカルインタフェースを追加します。

次を実行します。

「ファイル」--->「新規プロジェクト」
「一般」--->「Java アプリケーション」
そして、プロジェクトの名前を「tutorial」にします。

main 関数内に次を記述します。

    System.out.println("Hello World");
その後、それをコンパイルして実行します。
「構築」--->「主プロジェクトを構築」
「実行」--->「主プロジェクトを実行」

「Hello World」が、Ant によって出力されるほかの行とともに、出力ウィンドウに出力されるはずです。


グラフィカルインタフェースの編集:

次に、グラフィカルインタフェースエディタを使用します。これは、もっとも使用されている Java グラフィカルライブラリの 1 つである Swing に基づいています。

次を実行します。
「ファイル」--->「新規プロジェクト...」
「Java GUI フォーム」--->「JFrame フォーム」
そして、クラスの名前を「javaGui」にします。

「パレット」タブで「JLabel」を選択し、画面の中央に描画された JFrame 上にそれを配置します。「プロパティー」タブで、「text」変数に「Hello World」などの値を入力します。ここで時間を取ることをお勧めします。Swing は大きなライブラリですが、このグラフィカルエディタを使用すると、その機能のほとんどを利用できます。

アプリケーションを実行する前に、起動時に使用されるクラスが JFrame であることを確認する必要があります。メニューで「ファイル」--->「tutorial プロパティー...」--->「実行」--->「プロジェクトの実行」を選択し、主クラスのフィールドに「tutorial.javaGui」と入力するか、参照ボタンを使用します。このようにすると、アプリケーションの起動時に tutorial.javaGui の main メソッドが使用されます。

アプリケーションを起動し、「Hello World」ウィンドウを確認してください (見た目がひどいですが):

警告:
NetBeans のグラフィカルエディタでは、Java で提供されないグラフィカルライブラリがいくつか使用されています。その結果、NetBeans のグラフィカルエディタを使用して構築した将来のソフトウェアで、それらを提供する必要があります。ただし、Swing API の基本を知らない場合は、グラフィカルインタフェースエディタの使用はお勧めしません。最初は自分でグラフィカルインタフェースを記述することをお勧めします。


Tomcat の統合:

昨年、Web 開発が勃興しました。Java はこの環境で非常にうまく動作します。NetBeans には Tomcat Web アプリケーションサーバーが含まれており、容易に開発できます。このパートでは JSP とサーブレットを作成します。その目的は、Web テクノロジを理解することではなく、NetBeans 経由で Tomcat を使用する方法を理解することです。


Web プロジェクトを作成する:

次を実行します。


「ファイル」--->「新規プロジェクト」
「Web」プロジェクト、「Web アプリケーション」テンプレートの順に選択します。
プロジェクト名を TomcatTest にします

プロジェクトが作成され、Tomcat アプリケーションサーバーに追加されます。

JavaBean を編集する:

フォームのデータを操作する方法は、ほかにもあります。それらの 1 つが、Java クラスによって値を収集する JavaBean です。このクラスにはデザインパターンを適用する必要があります (値ごとに取得メソッドと設定メソッドを使用します)。

JavaBean を実行する場合:

  • Web プロジェクト上にカーソルを設定します
  • 「ファイル」--->「新規ファイル...」
  • 「Java クラス」--->「Java クラス」
  • この Java クラスの名前を「bean」に、そのパッケージ名を「beanPackage」にします。

サーブレットを編集する:

サーブレットとは、Web ページの作成に使用される Java クラスのことです。実は、JSP ページは、自動的にサーブレットに変換されたあと、コンパイルされます。

サーブレットを使用してフォームを作成します。

  • サーブレットを作成する場合:
  • 「ファイル」--->「新規ファイル...」
  • 「Web」メニュー--->「サーブレット」を選択します
  • サーブレットの名前を「destination_servlet」、パッケージの名前を「servletPackage」にします。

destination_servlet の内容は次のようにしてください。

destination_servlet.java

WEB-INF/web.xml を開くと、サーブレットがすでに設定され、動作の準備が整っていることがわかります。

そして、やはり JSP フォームを使用します。

send_servlet.jsp

「プロジェクト」タブでプロジェクトを右クリックし、「プロジェクトを実行」を選択します。Web ブラウザで index.jsp ファイルが開かれます。サーブレットベースのソリューションをテストするには、3 番目のリンクをクリックします。


タグを定義する:

タグファイルは、個人用のタグを定義するために使用されます。これを使用すると、ページヘッダーやページフッターを容易に定義できます。したがって、これを作成します。

個人用のタグを作成するには、次を実行します。
「ファイル」--->「新規ファイル...」
テンプレートの中から、「Web」--->「タグファイル」を選択します
ファイルの名前を「powered」にします

その他:

「実行時」タブでサブディレクトリ「サーバーレジストリ」を選択すると、Tomcat の一部の機能に直接アクセスできます。

ただし、Tomcat の開始と停止を除けば、その管理機能はやはり限定されています。ドキュメントを読み、構成を自分で変更することをお勧めします。起動時の時間が長くなるものの、パフォーマンスは向上します。
