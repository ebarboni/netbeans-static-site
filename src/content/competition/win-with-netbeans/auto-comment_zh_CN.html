NetBeans 4.0 中的炫酷自动注释工具 - 现在，文档编制既轻松又愉快！

撰稿人：Jason Stapley，与 NetBeans 共赢参赛作品

简介

抱歉，我必须得起一个让您忍俊不禁的文章标题，否则，您可能不会阅读有关文档编写的文章。
我一直很喜欢用 Sun 发布的 JDK 中的 JavaDoc 工具，它能为您代码的后续开发者/维护者指明正确的方向。或者，如果您和我一样，过了几个月、几周或几天就忘了代码的工作方式，该工具可以为您自己指明正确的方向。

自动注释工具可以替我完成哪些工作？

当然啦，它不会替您编写文档。您还得自己编写，不过，它以 JavaDoc 友好的方式设置注释格式和编排注释。
另外，还有一个重要的功能，它会标出缺少的注释。这提供了一种更自然的方法来验证 Java 注释。在开始使用 NetBeans 或运用自动注释工具之前，我得先运行 Javadoc 工具，并根据收到的警告更改注释或参数。这种方法既浪费时间又枯燥无味，同时也意味着出现错误的几率更高。而在使用自动注释工具时，将立即向您提醒错误或遗漏之处，您可以随后纠正这些错误，而无需始终返回到代码并进行更正。

我如何使用自动注释工具？

  1. 打开 Netbeans 项目
  2. 在“项目”窗口中，打开一个 Java 文件或选择 Java 文件或类节点。否则，将无法使用自动注释工具。能否使用自动注释工具完全基于上下文。如果为另一个类运行自动注释工具，该工具并不会打开另一个自动注释实例。这说明您每次只能编辑一个 Java 文件的注释。您不能将项目作为一个整体来运行自动注释（功能提示）
  3. 从“工具”菜单中，选择“自动注释…”

在打开自动注释工具后，将会在左窗格中看到类中的方法列表以及一系列具有图标的按钮。

这些按钮是过滤器。如果某个类包含很多方法和变量，自动注释过滤器可以帮助您将重点放在需要注释的内容上。

使用自动注释过滤器

单击左侧的第一个按钮。此按钮看起来像一个带有白色勾号的绿色框。这是一个切换按钮。在按下该按钮时，将显示所有符合 JavaDoc 规范的注释。

如果要查看更详细的过滤器按钮说明，请转到“帮助”菜单，然后单击“帮助内容”。接下来，单击“搜索”标签并输入“自动注释”关键字。或者，直接单击自动注释工具中的方法列表旁边的帮助按钮。

请注意过滤器按钮下方的类方法列表中的每个方法旁边显示的图标。

Java 注释正确图标

如果显示的图标看起来像带有白色勾号的绿色框，则说明该方法或变量的注释符合 JavaDoc 规范，您不需要完成额外的工作。您只需要检查是否输入了有关该方法或变量的相关/准确/最新的注释。

缺少注释图标

如果您看到的图标看起来像带有白色 X 的红色框，则表示方法注释不符合 JavaDoc 规范或没有任何关联的注释。当我最初开始使用 JavaDoc 时，我有时会编写类似下面的注释块... 

/*
* Java rocks
*/

当然啦，不会在 Java 文档中显示任何内容，因为缺少一个额外的星号。如果通过自动注释在“JavaDoc 注释文本：”文本字段中输入注释，自动注释将使用正确格式在代码中编写注释，例如：

/**
* Java rocks

*/

如果错误是“缺少注释”，自动注释将在方法列表下方的列表框中向您发出警告。列表下面显示的错误消息是“JavaDoc 注释缺失”。如果已注释方法或变量并标出了此错误，您可以单击“查看源代码”按钮以检查是否缺少额外的 *。重申一下，与自动注释工具中的所有内容一样，它也是完全基于上下文的。如果未选择方法，“查看源代码”按钮将灰显。

部分错误或完全错误的图标

我们最感兴趣的图标是部分错误图标，它看起来像一个带有雷电标志的三角形。大多数时候，如果具有参数的方法注释中不包含 @param 标记，则会看到此错误。自动注释告诉您哪个参数缺少标记，例如：缺少 <您的参数名称> 参数的标记。

另一个出现部分错误的例子是，在注释中添加了多余的标记。例如，您将 author 标记添加到类方法中，而不是添加到类中。请看下面的示例。 


/**
 * This method does something
 * @param Something
 * @Author Jason Stapley – This is an error. Not me, the Author tag!
 */
Void doSomething(String Something) {
    System.out.println(Something);
}

您可以在自动注释中更正这种类型的错误，方法是，在标记列表中选择多余的标记，然后单击“删除”按钮。这就引出了下一个主题“添加标记”。

在注释块中添加标记

在自动注释工具中，单击标记列表下面的“新建...”按钮，将会看到新的标记窗口。该窗口列出可以添加到注释块中的标记类型。最常见的标记是 @Param 和 @Return。在选择所需的标记类型后，如果这是一个参数，则必须在参数字段中命名该参数。该参数必须与方法中的参数相匹配。如果该参数有错误，请不要担心，自动注释会将该参数标出来。

标记顺序

您不必按任何定义的顺序添加标记以使 JavaDoc 正确无误。如果标记对您来说是合乎逻辑的，对阅读您的代码的其他人来说也是如此。通过使用自动注释，您可以更改注释块中的标记顺序。例如，如果您希望 return 标记始终是列表中的最后一个标记，请选择该标记，然后单击“下移”按钮以将其移到列表底部。如果您希望该标记位于列表顶部，请选择该标记，然后单击“上移”按钮。重申一下，只能在所执行的操作的上下文中使用这些按钮。

HTML 标记

由于 JavaDoc 创建 HTML，因此，您可以在注释中添加 HTML 标记。选择一个要注释的方法或字段，然后在“JavaDoc 注释文本：”字段中键入 <，等待几秒钟，将会看到一个标记列表，您可以就像编辑代码一样使用这些标记。您可以键入一个对象名称，然后键入一个圆点，将会看到一个方法列表，这也同样适用于标记。也可以在标记“说明”字段中使用自动完成功能。

还可以在自动注释屏幕右下角选择 JavaDoc 中的常用标记，这些标记是 <B> <I> <U> <CODE> <PRE> <@link>。

这是多强大的功能呀！

刷新自动注释

有时，您需要返回到代码。例如，您忘记在注释块开头添加两个星号，自动注释将找不到您的注释，因此，您必须单击“查看源代码”并检查代码。如果确实输错了内容，并在更正后返回到自动注释，最好单击“刷新”按钮以确保不会做重复的工作。
