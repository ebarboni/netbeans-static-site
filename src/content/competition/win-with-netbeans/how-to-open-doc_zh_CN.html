温馨提示：如何从 NetBeans 中打开任何类型的文档。

撰稿人：Vincent Brabant，与 NetBeans 共赢参赛作品

我如何让 NetBeans 使用 MS Word 打开 .doc 文档，使用 Excel 打开 .XLS 文档以及使用 Powerpoint 打开 .PPT 文档，等等。
在单击 .LNK（通常显示在桌面上）时，我如何让 NetBeans 打开相应的应用程序。

事实上，这并不难。

正如您所知道的一样，我们可以从 NetBeans 中打开 PDF 文件。

第一次单击 PDF 文件时，NetBeans 将显示下面的对话框：



您必须选择一个可执行文件。从逻辑上讲，我们要选择的是 Acrobat Reader，而不是市场上销售的任何其他 PDF 查看器。
但事实上，您可以任意选择所需的任何可执行文件。

此处实际是一个提示。

在 Windows 95、98 和 ME 中，命令目录中包含一个名为 Start.exe 的可执行文件。
该可执行文件可以将文件作为参数：任何类型的文件。它查看哪个应用程序与该文件的扩展名关联，然后启动该应用程序。


我们将在下面介绍如何实现此目的。

单击 "Select"（选择）并转至 Windows 目录，然后选择命令目录并搜索 start.exe。



选择该文件，然后单击 "Select"（选择）。当前查看器不再是 acroread，而是 start.exe 的绝对名称。



现在，将正常启动 Acrobat Reader 并显示所选的 PDF 文件。
这是如何实现的？start.exe 将 PDF 文件名作为参数，并查看哪个应用程序与该扩展名相关联。
由于我将 PDF 与 PC 上的 Acrobat Reader 相关联，因此，系统将打开 Acrobat Reader。

问题是，start.exe 仅在 Windows 95、98 和 ME 中存在。它在 Windows NT、2000 或 XP 中并不存在。
但在这些系统中，start 命令是众所周知的。它是作为命令处理器扩展实现的。
因此，要解决该问题，我们必须执行以下操作：
1. 在 C:\ 中创建一个 .BAT 文件。
该 BAT 文件将命名为 nbLauncher.bat。
该 BAT 文件包含以下内容：
start %1
2. 选择该 BAT 文件作为当前查看器。



但我如何打开其他文件，如 .DOC、.XLS、.EXE 甚至 .LNK 文件？

在主菜单中，选择 "Tools"（工具）| "Options"（选项）。
显示 "Option"（选项）面板。
展开 "IDE Configuration"（IDE 配置）| "System"（系统）| "Object Types"（对象类型），然后选择 "PDF Document Objects"（PDF 文档对象）。
在 "properties"（属性）面板中，单击 "Extension and Mime Types..."（扩展名和 MIME 类型...）。


缺省情况下，仅在项列表中显示 PDF，但您可以任意添加希望能够从 NetBeans 中启动的任何扩展名。
例如，我添加了 doc、xls、lnk 和 exe 后缀。


完成后，单击 "OK"（确定）.

然后单击 "Close"（关闭）。

现在，在 "Favorites"（收藏夹）窗口中添加一个目录以包含一些文档或可执行文件。另外，还要在 "Favorites"（收藏夹）窗口中添加您的桌面。
现在，您可以单击每个 .LNK 文件，将会启动相应的应用程序。
单击 .DOC 文件，将在 MS Word 中显示您的文档。

非常简单，这确实是非常有用的提示。
