NetBeans 使用数据跟踪

使用数据跟踪的目标是收集和分析有关 NetBeans 用户使用的高级功能的统计信息。通过更好地了解哪些是最常用和最不常用的功能，可以帮助我们更好地规划和设计将来的版本。收集的数据是严格匿名的，并自动发送到 netbeans.org 服务器。数据收集和提交不会影响性能，也不会影响正常的工作流程。

收集的使用数据类型

  • IDE 配置 - 版本号、操作系统信息、启用的插件列表
  • 使用的项目类型（如 Java SE、Java SE 自由格式、Java Web 应用程序、Java 企业应用程序、Ruby/Rails 应用程序、PHP、C/C++、NB 模块项目等）
  • 使用的 Web 应用程序框架（如 Struts、JSF、Visual JSF、Spring、Hibernate 等）
  • 部署时使用的服务器和运行环境类型和版本（如 GlassFish、Tomcat、WebSphere、JBoss、WebLogic、WEBrick 等）
  • 连接到数据库时使用的 JDBC 驱动程序类型（如 MySQL、JavaDB、Oracle 等）
  • 使用的特定生产力功能（如调试器、Profiler、GUI 生成器）
  • 用户使用的版本控制系统类型（CVS、Subversion、Mercurial、ClearCase 等）
  • 访问的帮助主题。

不会收集的数据

  • 用户名和口令。
  • 路径、文件名和类名。
  • 用户的源代码片段。

NetBeans 团队非常感谢您参与使用数据跟踪计划！
