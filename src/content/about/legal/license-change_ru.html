Двойное лицензирование среды IDE NetBeans 6.0 согласно CDDL и GPLv2 с исключением Claspath

С момента запуска платформы NetBeans.org в 2000 лицензирование исходного кода NetBeans изменилось от Общей лицензии Sun (Sun Public License) до Общей лицензии разработки и распространения (Common Development and Distribution License, CDDL) открытого исходного кода. С выпуском бета-версии среды IDE NetBeans 6.0 мы добавили Общую лицензию GNU v.2 (GNU Public License v.2, GPLv2) к уже существующей CDDL. Большая часть кода среды IDE NetBeans 6.0 доступна по двойной лицензии, состоящей из CDDL v1.0 и GPLv2.

Зачем нужна смена лицензии?

  • Лицензия GPL v2.0 предоставляет дополнительную возможность поставщикам, которые не могут работать в NetBeans с лицензией CDDL.
  • Добавление GPLv2 как варианта лицензии содействует более комфортной работе с Linux на платформе NetBeans.
  • Добавление GPLv2 с исключением Classpath поможет сохранить единообразие портфелей продукта и пакетов в NetBeans. Sun открыл исходный код внедрения JDK по GPLv2, а проект GlassFish имеет двойную лицензию: CDDL и GPLv2 с исключением Classpath.

CDDL и GPLv2 — это лицензии, утвержденные OSI, которые соответствуют требованиям определения открытого исходного кода.

Последствия для разработчиков

Смена лицензий не должна существенным образом повлиять на проекты с применением программного обеспечения NetBeans или разработчиков, создающих код программного обеспечения, интегрированного с NetBeans. Упоминание лицензии, которое необходимо добавить в исходные файлы — вот единственное изменение, способное повлиять на повседневный труд разработчика.

Часто задаваемые вопросы по двойной лицензии:

В: Что такое смена лицензии?
О: Sun представляет GPLv2 с исключением Classpath для NetBeans как вторую возможность лицензирования вместе с CDDL. GPL v2 — это лицензия для открытого исходного кода. Прочитайте текст лицензии.

В: Почему Sun требует двойной лицензии для NetBeans — CDDL и GPLv2?
О:

  • Лицензия GPL v2.0 предоставляет дополнительную возможность поставщикам, которые не могут работать в NetBeans с лицензией CDDL.
  • Добавление GPLv2 как варианта лицензии содействует более комфортной работе с Linux на платформе NetBeans.
  • Добавление GPLv2 с исключением Classpath поможет сохранить единообразие портфелей продукта и пакетов в NetBeans. Sun открыл исходный код внедрения JDK по GPLv2, а проект GlassFish имеет двойную лицензию: CDDL и GPLv2 с исключением Classpath.

В: Что такое исключение Classpath?
О: Исключение Classpath было разработано в Free Software Foundation GNU/проекте Classpath. Исключение позволяет связывать приложение, доступное по любой лицензии с библиотекой, являющейся частью программного обеспечения с GPL v2, при этом приложение, являющееся предметом GPL, не должно быть открыто в соответствии с GPL.

В: Зачем нужно исключение Classpath?
О: Если приложение должно распространяться с компонентами NetBeans, которые лицензированы GPL v2, это приложение может работать в соответствии с требованиями GPL, согласно которым код, поставляемый как часть "работы на базе программы [GPL]", должен соответствовать условиям лицензии GPL. Соответственно, требуется исключение к лицензии GPL, которое исключает из требований лицензии любое приложение, которое связано с реализацией GPL. Исключение Classpath предоставляет такую возможность. Это может быть важно для разработчиков модулей, которые всегда связаны с API-интерфейсами NetBeans, а также для разработчиков приложений на платформе NetBeans.

В: Как добавление GPL v2 повлияет на текущие дистрибутивы?
О: Лицензия не повлияет на них. Текущие и будущие дистрибутивы будут по-прежнему доступны по CDDL. Новые дистрибутивы, начиная с NetBeans 6.0, будут одновременно доступны по CDDL и GPLv2 с исключением Classpath.

В: Как можно выпускать что-либо с двумя лицензиями?
О: Двойное лицензирование — это распространение одного и того же программного обеспечения с двумя (или несколькими) наборами условий. Если программное обеспечение подпадает под две лицензии, получатели могут выбрать условия, с которыми они получат это программное обеспечение. Причинами для перехода к двойному лицензированию являются среди прочего бизнес-модели и совместимость лицензии. В случае NetBeans код распространяется с двумя лицензиями, CDDL и GPLv2 с исключением Classpath, для совместимости лицензии. Это позволит коду из проектов бесплатного программного обеспечения с различными лицензиями быть совместимым с NetBeans и даст нашим пользователям возможность выбора лицензии, которая больше соответствует их потребностям. Корпорация Sun добавила лицензию GPLv2 в качестве варианта, поскольку по соображениям политики Sun никогда не отбирает права, так что код NetBeans будет оставаться доступным по лицензии CDDL.
