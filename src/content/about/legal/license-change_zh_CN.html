依照 CDDL 和 GPLv2（类路径例外）为 NetBeans IDE 6.0 授予双重许可

自 2000 年推出 NetBeans.org 以来，NetBeans 源代码许可证已从开源 Sun 公共许可证演变为开源通用开发和分发许可证。随着 NetBeans IDE 6.0 测试版的发布，我们在已采用的 CDDL 基础上添加了 GNU 公共许可证 v2。绝大部分的 NetBeans IDE 6.0 代码是依照双许可证（包括通用开发和分发许可证 (CDDL) v1.0 和 GNU 通用公共许可证 (GPL) v2））提供的。

为什么要改变许可证？

  • GPLv2.0 许可证为无法依照 CDDL 许可证使用 NetBeans 的供应商提供了一个额外选项。
  • 通过将 GPLv2 添加为许可证选项，可以使 NetBeans 甚至更加 Linux 友好。
  • 通过将 GPLv2（类路径例外）添加到 NetBeans 中，可以使整套产品和包保持一致。Sun 依照 GPLv2 提供开源 JDK 实现，并依照 CDDL 和 GPLv2（类路径例外）为 GlassFish 项目授予双重许可。

CDDL 和 GPLv2 都是 OSI 批准的许可证，它们满足开源定义的要求。

对开发者的影响

我们预计这种改变不会对使用 NetBeans 软件的项目或编写与 NetBeans 集成在一起的软件的开发者造成很大的影响。源文件中所需的许可声明将发生变化 - 这是唯一可能影响开发者日常工作的部分。

双许可证简明常见问题解答：

问：许可发生了什么变化？
答：Sun 推出了 GPLv2（类路径例外）作为 CDDL 以外的 NetBeans 许可证选项。GPLv2 是一个开源许可证。请参见许可证文本。

问：Sun 为什么要依照 CDDL 和 GPLv2 为 NetBeans 授予双重许可？
答：

  • GPLv2.0 许可证为无法依照 CDDL 许可证使用 NetBeans 的供应商提供了一个额外选项。
  • 通过将 GPLv2 添加为许可证选项，可以使 NetBeans 甚至更加 Linux 友好。
  • 通过将 GPLv2（类路径例外）添加到 NetBeans 中，可以使整套产品和包保持一致。Sun 依照 GPLv2 提供开源 JDK 实现，并依照 CDDL 和 GPLv2（类路径例外）为 GlassFish 项目授予双重许可。

问：什么是类路径例外？
答：类路径例外是由自由软件基金会的 GNU/类路径项目制订的。它允许您将依照任何许可证提供的应用程序链接到依照 GPLv2 许可的软件中包含的库，而该应用程序不受 GPL 要求公开其本身的限制。

问：为什么需要使用类路径例外？
答：如果需要依照 GPLv2 将应用程序作为 NetBeans 的一部分进行分发，该应用程序可能需要满足以下 GPL 要求：作为“基于 [GPL] 程序的作品”的一部分提供的所有代码还应获得 GPL 许可。因此，需要指定 GPL 许可证例外情况，以便明确将链接到 GPL 实现的任何应用程序从该许可要求中排除。类路径例外实现了这一目的。例如，这不仅对始终链接到 NetBeans API 的模块开发者非常重要，而且对在 NetBeans 平台上构建应用程序的开发者也是至关重要的。

问：添加 GPLv2 对当前分发有何影响？
答：这不会影响当前分发。现在和将来的分发仍然依照 CDDL 提供。从 NetBeans 6.0 开始，新的分发将依照 CDDL 和 GPLv2（类路径例外）提供。

问：如何依照两个许可证发布产品？
答：双重许可是依照两套（或更多套）不同的条款和条件分发相同软件的作法。在为软件授予双重许可时，接收人可以选择他们希望依照哪种条款获得软件。使用双重许可的两个常见动机是遵循商业模式和保持许可证兼容性。对于 NetBeans，我们使用 CDDL 和 GPLv2（类路径例外）这两个许可证分发代码以保持许可证兼容性。这样，就可以将以不同方式许可的免费软件项目的代码与 NetBeans 合并在一起，并允许我们的用户选择更适合他们的许可证。Sun 将 GPLv2 添加为一个选项，因为从政策上讲，Sun 绝不会剥夺用户的权利，因此，可以继续依照 CDDL 许可证提供 NetBeans 代码。
